Queue Contract & Semantics

SINGLE SOURCE OF TRUTH (SOT)

Canonical Semantics

Pomai Queue is a deterministic, bounded, crash-safe backend task queue.

Delivery & Ordering

  • At-least-once delivery: Guaranteed for pull consumers.
  • Ordering: Strict per queue + consumer group by persisted message offset.
  • Lease Rules: Ack is only valid for currently IN_FLIGHT messages with a matching lease token.

Message States

READY
→
IN_FLIGHT
→
ACKED
/
DEAD

A message is in exactly one state. No dual-state membership is allowed.

System Invariants

The correctness of the system is proven by the following invariants:

  1. A message is in exactly one state (READY, IN_FLIGHT, ACKED, DEAD).
  2. Queue/group transitions are monotonic by sequence in the event log.
  3. IN_FLIGHT transitions must carry lease metadata and deadline scheduler registration.
  4. Recovery starts from durable segment logs and replays transitions deterministically.
  5. Retry budget overflow transitions strictly to DEAD.
  6. Scheduler work is bounded by the current wheel bucket, not full map scans.

Failure Modes & Recovery

The system is designed to handle failures gracefully without data corruption.

  • Crash during enqueue: Segment recovery truncates partial tail records; no half-record is replayed.
  • Crash with inflight: Entries are recovered as ready-for-redelivery, preserving at-least-once semantics.
  • Corrupted State: Engine explicitly returns corruption/IO status rather than allowing silent divergence.
