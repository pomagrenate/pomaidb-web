Testing & Reliability Guide

Methodologies for deterministic verification, crash resilience, and stress testing.

Determinism Rules
  • Time: Uses EngineOptions::now_fn (FakeClock), never sleeps.
  • Randomness: Crash and randomized tests use fixed seeds.
  • Bounds: Workloads are bounded by fixed iteration counts.

1. Unit & Integration Tests

Execute the full suite using CMake CTest:

cmake -S . -B build
cmake --build build -j
ctest --test-dir build --output-on-failure
  • test_record: Encoding, CRC validation, corruption detection.
  • test_segment_log: Recovery and truncation boundaries.
  • test_queue_engine: State transitions, idempotency, fake-clock.
  • test_engine_integration: Producer/consumer accounting, restart checks.

2. Crash Oracle Harness

Exercises specific windows of vulnerability (Failpoints) to ensure atomicity.

# Fast CI Sample
./build/crash_runner 2

# Deep Verification
./build/crash_runner 20

Failpoints: before_log_append, before_lease_issue, before_ack_transition, before_dlq_move, etc.

3. Stress Testing

Runs bounded workloads with concurrent producers/consumers and checks invariants.

./build/pomaiqueue_stress --seconds 10 --producers 4 --consumers 4 --payload_bytes 1024 --queues 2

Outputs JSON throughput/latency summary and exits non-zero on invariant violation.
